Runner Requirements

Install the Textual dependency once inside the unzipped run bundle, then launch the runner.

python3 -m pip install -r requirements.txt
./run_all.sh

Pipeline Runtime

The pipeline itself requires Nextflow and a container runtime such as Singularity. For Google runs, set the configured Google service account path and Tower token.

conda install bioconda::nextflow
export GOOGLE_APPLICATION_CREDENTIALS=/path/to/pipeline-service-key.json
export TOWER_ACCESS_TOKEN=your_token_here

How To Use The Configurator

1. Configure

Upload a base `nextflow.config`, set the output directory, credentials path, repository branch, and one row per sample. Use forced columns when samples should share the same parameter value.

2. Export And Run

Click Export Run, unzip the bundle, install requirements, and launch the Textual runner.

unzip pipeline_run_*.zip
cd pipeline_run_*
python3 -m pip install -r requirements.txt
./run_all.sh

3. Resume Or Inspect

The runner stores durable status in `runner_state.json` and archived logs in `logs/<sample>/`, so you can close and reopen the interface without losing run state.

python3 pipeline_runner.py --status
python3 pipeline_runner.py --tail Sample_1
python3 pipeline_runner.py --resume Sample_1
